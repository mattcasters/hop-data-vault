Minimal valid Apache Hop file skeletons (topology reference only).

Pipeline (.hpl):
- Root <pipeline> with <info> (name, name_sync_with_filename=Y, parameters).
- <order> contains <hop> elements: <from>, <to>, <enabled>Y</enabled>.
- Each <transform> has <name>, <type> (plugin XML type), and <GUI><xloc>/<yloc>.

Workflow (.hwf):
- Root <workflow> with <name>, <name_sync_with_filename=Y, <parameters>.
- <actions> contains <action> nodes with <name>, <type>, <xloc>, <yloc>.
- <hops> connects actions with <from>, <to>, <enabled>, <evaluation>, <unconditional>.

When proposing ADD_TRANSFORM or ADD_ACTION, use transformPluginId/actionPluginId from the
plugin catalog JSON. New nodes receive default plugin metadata only — no config parameters.
