Apache Hop production standards (advisory in Hop GUI AI Help — user may override).

Layout (topology proposals):
- Snap transform/action locations to a 32px grid.
- Prefer horizontal spacing of 192px or 224px between connected nodes.
- Prefer vertical row spacing of 128px or 160px for S-curve or U-curve flows.
- Avoid crossing hops; branch cleanly up or down before running parallel streams.

Naming (topology proposals):
- Transform and action labels: maximum 20 characters.
- Do not prefix names with transform/action types (icons already show type).
- Prefer short verbs: Read, Prep, Write, Get, Log for entry/exit steps.
- Intermediate steps should describe the business rule (e.g. "Keep valid parents").

Complexity (advisory):
- Prefer fewer than 20 transforms per pipeline and 15 actions per workflow.
- Split large flows into sub-pipelines or sub-workflows when proposing many nodes.

Notepads (optional ADD_*_NOTE proposals):
- Suggest #Story, #HowToTest, and #Author header notepads at yloc=0 when creating new files.
- Include "Hop AI Help (AI Assistant)" in #Author when AI proposes structural edits.

Parameterization (advisory only — do not set paths in proposals):
- Prefer ${PROJECT_HOME} for file references in advice text; never emit connection strings or SQL.

Out of scope for hop_proposals:
- Transform/action configuration (SQL, fields, connections, distribute tags, routing targets).
- Plugin dialog settings. Propose topology only; user configures details manually.
